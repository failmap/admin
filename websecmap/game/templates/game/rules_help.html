Rules

This page outlines some help / rules for joining this session.

Rule 1: Quality over quantity

Having a great dataset will increase the likelyhood of this tool being taken serious. It also makes the downloadable datadumps of failmap a pleasure to use. Please take care that you're adding information that is genuine and of quality. To assist you, there are pages that list already known organizations and known urls: you can CTRL+F through it.

Rule 2: Be excellent to each other

Please help each other and try to answer each others questions.

Rule 3: Don't cause abuse

Make sure that you're not abusing online services or online infrastructure in your quest to find more information. Usually there is a better way to find what you're looking for.

Help

What kind of organizations can i add?

There are various organization types, which all fall under "governmental" or "vital infrastructure".

You can add all governmental organizations and "agencies" that are exclusively government appointed.

Do not submit urls of non-governmental organizations such as 3rd party companies that deliver services to the government: for example, do not add the url of the webdesigner for the organization.

Do not add urls of specific people such as linked-in pages.

What can i do if i find a serious vulnerability

Responsible disclosure. Read this: https://www.ncsc.nl/english/current-topics/news/responsible-disclosure-guideline.html

Can i use automated submissions: yes, and be warned.

Yes, you can submit urls automatically. But do make sure that you're taking into account using the CSRF value and make sure your submissions are quality. Any submission that is not of quality will cost someone a lot of time validating. Be sure to check first, especially on wildcard domains.

Be warned though: if you submit too much rubbish data, you / your team get kicked and all submitted data baleeted.

What happens after something is added?

For each url that has been added, an onboarding process starts. This process consists of the following steps:

  1. Trying to contact the url: finding endpoints of various services.
  2. Perform tests on the endpoints.
  3. If url is top level: Automatically determine if there are more subdomains.
  4. Update the report on the website.

Where can i download all submitted information?

The filtered, higher quality, dataset can be downloaded from the failmap website we're currently working for. Look at the bottom of the page at datasets. You'll find ready made datasets for various purposes.

Where can i find more domains / subdomains?

There are many sources for subdomains:

  • Various online services index subdomains and may have some unique ones.
  • If you have access to DNS servers or services, you might be able to get subdomains from that.
  • Certificate transparency databases such as crt.sh have long lists of subdomains and domains for which TLS certificates are requested. Note that some of these may be outdated, expired and will miss all sites that do not use encryption (no encryption = lots of points).
  • Tools like theHarvester and DNSrecon can help you find subdomains: this is also what this website does after onboarding: use various standard tools to find subdomains in a friendly way.
  • Crawling the site, or searching using smart search queries on the larger search engines.
